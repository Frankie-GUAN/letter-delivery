## 一、产品定位

**一句话**：用相机在真实世界留下信，让别人走到那个地方才能看见。

**核心理念**：相机不再只是拍照工具，而是一扇窗户——透过取景器，你看到的不只是眼前的真实场景，还有别人在此地留下的情感痕迹。每一封信都锚定在一个真实的GPS坐标上，只有走近那个地方、举起相机、对准那个角度，信才会从虚空中浮现。

**核心循环**：

```
发现 → 在罗盘地图上找到附近的信
对齐 → 打开相机，对准原始拍摄角度（公开信可选）
阅读 → 信浮现，文字+照片+回响
写信 → 在同一地点拍下你的回信或新信
分享 → 密信生成分享口令，发给特定的人
```

## 二、三种信件

### 2.1 公开信 — 众人的到此一游

在旅游景点、咖啡馆、街头转角留下你的信。后来的陌生人走到这里，打开相机就能发现。

**发信流程**：
```
打开相机 → 对焦当下场景 → 拍照 → 附上留言
→ 标记为「公开」→ 投放到当前位置 → 信件锚定在GPS坐标上
```

**收信流程**：
```
打开罗盘地图 → 看到附近有信封图标浮动
→ 走近目标范围（半径20m内）→ 切换到相机取景器
→ 画面中出现模糊的虚影信封 → 对准当时拍摄角度
→ 相似度上升，信封逐渐清晰 → 完全对齐后，信纸展开
→ 阅读内容 + 看到当时的参考照片 → 可以留下回复
```

**对齐开关**：默认开启。发信人可以选择关闭，关闭后任何人进入范围直接可见。

---

### 2.2 给自己的信（时光胶囊）— 未来的自己

在某个地方，拍下此刻的自己，写一段给未来的话。到了设定的时间，它会提醒你回来打开。

**埋下流程**：
```
拍一张照片 → 写上想对未来的自己说的话
→ 设置解锁时间（某年某月某日 / 倒数天数）
→ 可选：邀请好友一起「共埋」，对方也会在这个时间收到提醒
→ 可选：开启「对齐解锁」— 必须回到原地才能打开
→ 埋入当前位置
```

**打开流程**：
```
到达解锁时间 → 收到本地通知
→ 如果开启对齐：必须回到原地 + 对准原角度 → 信封浮现
→ 如果关闭对齐：任何地方都可以打开
→ 如果多人共埋：所有人都打开后，可以看到彼此当时的留言
```

---

### 2.3 密信 — 两个人的秘密

课桌下的留言、只有你们知道的角落。一封信指定一个人，生成口令分享给Ta，Ta循着线索才能找到。

**发信流程**：
```
选一个地方 → 拍下「线索照片」（不一定拍全貌，可以只是一角）
→ 写留言 → 选择「指定接收人」
→ 生成分享口令 → 发送给Ta
```

**收信流程**：
```
收到口令 → 打开作品 → 输入口令 → 获得线索照片 + 大致范围
→ 根据线索找到那个地方 → 进入范围后，直接看到信（默认无对齐）
→ 阅读 → 可选：就地回复或留下新的密信
```

```
┌──────────────────────┐
│  ＜ 返回地图          │
├──────────────────────┤
│                      │
│    📍 GPS锁定指示    │
│                      │
│         ✉️           │  ← 悬浮信封（AR标签）
│      ┌──────┐        │
│      │ 信纸  │        │
│      └──┬───┘        │
│         │ 锚定线     │
│     ════╧════       │  ← 地面参考线
│                      │
│  [🔍 对齐 ███░░░░░]  │  ← 对齐进度条
│                      │
│  [拍照写信] [对焦]   │
└──────────────────────┘
```

**信封行为**：
- 离目标 >20m：不显示
- 10-20m：小光点晃动，方向指示
- <10m：信封浮现，半透明
  - 开启对齐时：模糊虚影，随相似度升高变清晰
  - 关闭对齐时：直接可见
- 对齐成功：信封翻开封口，纸页展开动画

**无信时可做的事**：底部按钮变为 `[在此留下第一封信]`，任何地点任何人都能投下第一封信。

## 四、AI场景匹配引擎

### 4.1 对齐机制

```
原始照片（发信时拍摄）
     ↓
特征提取引擎（Canvas处理）
     ↓
特征向量（256维，存入信数据）
     ↓
    ... 收信人到达附近后 ...
     ↓
当前取景帧（每秒采样一帧）
     ↓
同算法特征提取
     ↓
余弦相似度比对 → 0-100%对齐得分
     ↓
驱动取景器中的对齐进度条 + 信封清晰度
```

**视觉反馈梯度**：

| 得分区间 | 信封表现 | 可读性 |
|---------|---------|-------|
| 0-30% | 虚线轮廓，光晕闪烁 | 不可读 |
| 30-60% | 轮廓清晰，浮现头像 | 标题模糊可见 |
| 60-90% | 信纸徐徐展开 | 文字逐渐可辨 |
| 90-100% | 完全清晰，照片浮现 | 完全可读，可回复 |

### 4.2 特征提取算法

采用纯前端传统CV方案（无深度学习依赖，Demo版可用）：

```
1. 缩放图像至 256×256
2. 分 4×4 网格，每格提取：
   - 平均色相/饱和度（HSV空间）
   - 边缘方向直方图（Sobel → 8方向）
   - 亮度梯度（Laplacian）
3. 拼接为 256 维特征向量
4. 余弦相似度 → 映射为对齐进度
```

**性能预估**：单帧提取 <50ms，每秒采样一帧，实时可用。

### 4.3 AI信件润色（完整版）

完整版接入大模型API，用户输入大白话 → AI按选定情绪（温柔/俏皮/深情/怀念）生成氛围感短笺。Demo版降级为本地预设模板库。

## 六、完整版 vs Demo版

| 维度 | 完整版 | Demo版（互动空间） |
|------|--------|-------------------|
| 地图 | 高德/腾讯JS SDK | Leaflet + 预缓存瓦片 |
| AI对齐 | ✅ 完整特征提取+实时比对 | ✅ 相同（纯前端） |
| AI润色 | 大模型API | 本地预设模板库 |
| 数据同步 | 服务端同步 | IndexedDB 本地 |
| 密信口令 | 服务端解析+推送 | 本地匹配 |
| 通知 | 推送通知 | 本地定时检查 |
| 包体 | 无限制 | ≤8MB |
| 网络 | 需要 | 不需要 |

**核心保留原则**：取景器 + 对齐解锁的灵魂体验，完整版和Demo版完全一致。

## 八、约束与合规

- 完整版无包体/网络限制
- Demo版 ≤8MB，纯离线，无外部资源引用
- 内容不得包含违法违规、IP侵权内容
- 不涉及青少年不适宜内容
- 所有用户生成内容（UGC）本地存储，不上传服务器（Demo版）
