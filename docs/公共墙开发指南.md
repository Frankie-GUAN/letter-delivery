## 一、开发目标

公共墙是产品的**核心体验入口**。用户打开小程序，第一眼看到的就是一面旧公寓的墙，墙上散布着陌生人留下的信。

### 完整用户路径

```
打开App → 看到墙上的信封 → 点击一封 → 阅读 → 留下回响
                                              ↓
                         点击FAB按钮 → 选藏身方式 → 选信纸 → 写正文 → 署名 → 寄出 → 信出现在墙上
                                                                                ↓
                                                                      刷新墙 → 自己的信也在上面
```

### 2.4 时间状态

墙的视觉效果应随 `WallEngine.getCurrentTimeState()` 变化：

- **清晨 (5:00-9:59)**：偏冷灰，少量"露水"光斑
- **正午 (10:00-15:59)**：偏暖白，对比度较高，信封阴影清晰
- **傍晚 (16:00-19:59)**：暖黄/琥珀色，长阴影，最温暖的时段
- **深夜 (20:00-4:59)**：深蓝灰，只有"路灯"光晕照亮局部

实现方式：在 `.wall-surface` 上叠加 CSS `background` 渐变 + 半透明 overlay。

### 2.5 预置信件注入

首次使用时，将 `PresetLetters.ALL` 注入 Storage：

```js
function loadLetters() {
    let stored = Storage.getLetters();
    const publicLetters = stored.filter(l => l.type === 'public');

    if (publicLetters.length === 0 && typeof PresetLetters !== 'undefined') {
        PresetLetters.ALL.forEach(l => {
            l.id = generateId();
            l.type = 'public';
            Storage.saveLetter(l);
        });
        stored = Storage.getLetters();
    }

    return stored.filter(l => l.type === 'public');
}
```

桩代码已实现此逻辑。

## 三、关键技术细节

### 3.1 信封位置计算

避免信封重叠：在 `calculateEnvelopePositions` 中使用简单的网格扰动算法，而非纯随机。桩代码当前使用纯随机 + slot 基础偏移，基本可用。若发现重叠严重，改为：

```
将墙面分为 N 行 × 3 列的网格
每个网格内随机偏移 ±15%
同一行最多放3个信封
```

### 3.2 模态层状态管理

关键原则：**关闭模态时必须清空所有临时状态**。`closeModal()` 已在桩代码中实现。每次写信/读信前检查 `$modalOverlay.classList.contains('hidden')`。

### 3.3 草稿自动保存

写信过程中，如果用户关闭模态而未寄出：
- 在关闭前检查 textarea 是否有内容
- 如果有 → `Storage.saveDraft({ id, content, paperType, slot, signature, updatedAt })`
- 下次打开写信面板时，提示"你有一封未完成的信，要继续写吗？"

### 3.4 回响投递反馈

当前桩代码的 `showEchoFeedback` 实现了一个 emoji 飘起动画。需要补充：
- emoji 飘起后，风铃数量实时刷新（已实现）
- 如果该信收到了所有5种回响 → 出现特殊视觉（如信纸边缘微微发光）
