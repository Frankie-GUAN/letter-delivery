## 一、你需要完成的功能

### 功能 1：公告栏渲染

**触发时机：** 用户切换到公告栏（`TimeLetter.show()` 被调用时）

**要做什么：**
1. 从 `Storage.getTimeLetters()` 读取所有时光信
2. 调用 `WallEngine.renderTimeBoard(container, timeLetters, onClick)` 渲染
3. `onClick` 回调里调用 `handleTimeLetterClick(timeLetter)` —— 这个函数已定义好入口，你需要完善里面的三种状态判断

**注意：** 桩代码中 `WallEngine.renderTimeBoard` 已实现，直接用即可。

---

### 功能 2：创建时光信（三步流程）

这是模块最核心的交互。入口是 `startCreating()`，桩代码已写好三步流程的骨架：

#### 步骤1：写信 + 设定打开时间

```
用户操作：
  1. 输入正文（最长500字）
  2. 输入署名
  3. 选择打开时间（<input type="datetime-local">）
  4. 点击"下一步：拍照"

你需要做的：
  1. 校验正文不为空
  2. 校验时间 > 当前时间
  3. 把 { content, signature, openTime } 暂存到 state.creating
  4. 调用 step2TakePhoto()
```

#### 步骤2：拍照

```
用户操作：
  1. 点击拍照按钮
  2. 对准想"埋信"的地点拍照

你需要做的：
  1. 调用 Camera.takePhoto() 获取照片 dataURL
  2. 调用 Camera.extractFeatures(dataURL) 提取视觉特征
  3. 调用 Camera.makeFadedPhoto(dataURL) 生成褪色预览图
  4. 把 { photo, features, fadedPhoto } 合并到 state.creating
  5. 调用 showStep3Preview()
```

**异常处理：** 拍照取消或失败 → 提示"拍照失败，请重试"，不关闭模态层。

#### 步骤3：预览 + 确认

```
用户操作：
  1. 看到褪色照片 + 信纸缩略 + 倒计时
  2. 点击"埋下这封信"确认 / 点击"重新拍照"返回步骤2

你需要做的：
  确认 → 构建 TimeLetter 对象 → Storage.saveTimeLetter()
       → Storage.incrementStat('timeLettersCreated')
       → 关闭模态 → 刷新公告栏
  重新拍照 → 回到 step2TakePhoto()
```

**构建 TimeLetter 对象的关键代码：**

```js
const timeLetter = {
    id: 'TL_' + Date.now() + '_' + Math.random().toString(36).substring(2, 8),
    content: state.creating.content,
    signature: state.creating.signature,
    photo: state.creating.photo,
    fadedPhoto: state.creating.fadedPhoto,
    features: state.creating.features,
    openTime: state.creating.openTime,
    createdAt: Date.now(),
    unlocked: false,
    unlockPhoto: null,
    unlockedAt: null
};
```

### 功能 3：点击时光信 → 三种状态处理

`handleTimeLetterClick(timeLetter)` 已定义入口，你需要完善三个分支：

#### 状态A：时间未到 + 未解锁

```
显示内容：
  - 褪色照片（Opacity: 0.6）
  - "🔒 尚未解锁"
  - 倒计时文字（X天X小时后可打开）
  - "届时请到拍照的地点打卡"
```

**倒计时函数**（可复用桩代码中的 `countdownText(ms)`）：
```js
function countdownText(ms) {
    if (ms <= 0) return '现在';
    const days = Math.floor(ms / (1000 * 60 * 60 * 24));
    const hours = Math.floor((ms % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60));
    if (days > 0) return `${days}天${hours}小时`;
    const mins = Math.floor((ms % (1000 * 60 * 60)) / (1000 * 60));
    return `${hours}小时${mins}分钟`;
}
```

#### 状态B：时间已到 + 未解锁（需要拍照解锁）

```
显示内容：
  - 褪色参考照片（Opacity: 0.5）
  - "时间到了！请站在原来的位置，用同样的角度拍一张照。"
  - [📷 拍照解锁] 按钮

拍照后：
  1. Camera.takePhoto() 获取照片
  2. Camera.extractFeatures() 提取特征
  3. Camera.compareFeatures(原始特征, 新特征) 得到相似度 (0~1)

  如果相似度 >= 0.6：
    → Storage.unlockTimeLetter(id, newPhoto)
    → 跳转到状态C（已解锁视图）

  如果相似度 < 0.6：
    → 显示角度提示 + 当前匹配度百分比
    → 用户可以重试拍照
```

**角度提示逻辑：** 比较原始特征和新特征的5区亮度差异，给出方向性提示：
```js
function generateAngleHints(original, current) {
    // 比较各区亮度，找出差异最大的区域
    // "上方太亮了，稍微低一点"
    // "右边太亮了，往左转一点"
    // "试着调整一下角度，找到和原来差不多的构图"
}
```

#### 状态C：已解锁

```
显示内容（核心亮点！）：
  - 左右两张照片并置：
    左边：埋信时的褪色照片 + "2026.10.3 · 埋信时"
    右边：解锁时的新照片 + "2027.1.1 · 打开时"
  - 中间：信的内容（使用 LetterComponent.renderContent()）
  - 底部：一行 AI 观察文字
```

**AI 观察文字的生成：** 由于 Demo 阶段无网络，用本地模板拼接：
```js
function generateUnlockNote(originalFeatures, unlockFeatures, originalTime, unlockTime) {
    const days = Math.floor((unlockTime - originalTime) / (1000 * 60 * 60 * 24));

    // 比较亮度变化（光线方向变了）
    const centerDiff = Math.abs((originalFeatures.center || 0) - (unlockFeatures.center || 0));

    // 比较整体亮度（季节/时间变化）
    const avgOrig = (originalFeatures.tl + originalFeatures.tr + originalFeatures.bl + originalFeatures.br) / 4;
    const avgUnlock = (unlockFeatures.tl + unlockFeatures.tr + unlockFeatures.bl + unlockFeatures.br) / 4;

    let note = '';

    if (centerDiff > 30) {
        note = '光的方向变了。时间不是一条直线，它是有角度的。';
    } else if (Math.abs(avgOrig - avgUnlock) > 20) {
        note = avgUnlock > avgOrig
            ? `过了${days}天，这里好像更亮了。`
            : `过了${days}天，光线暗了些。可能是云，可能是季节。`;
    } else {
        note = `两张照片之间隔了${days}天。光没变，但看光的人变了。`;
    }

    return note;
}
```

## 二、UI 交互细节

### 模态层使用

所有弹窗使用全局 `#modal-overlay` + `#modal-content`（传入 init 时已绑定）。
关闭模态：点击 ✕ 按钮 / 点击背景遮罩。

桩代码中已有 `closeModal()` 方法。

### 按钮状态管理

- 拍照按钮在照片处理期间设为 disabled，防止重复点击
- "埋下这封信"按钮在点击后立即 disabled，防止重复提交

### 照片处理时的 loading 提示

```js
// 拍照处理期间显示一个简单的 loading
// Camera 的处理在 Canvas 上，不需要实际网络请求
// 但 extractFeatures 对较大图片可能耗时，加一个文字提示：
document.getElementById('tl-next-btn').textContent = '处理中...';
document.getElementById('tl-next-btn').disabled = true;
// 处理完后恢复
```
