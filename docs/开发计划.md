## 三、沟通约定

1. **共享层修改**：`shared/` 目录下的任何改动，必须先在群里说，得到对方确认后再改
2. **新增依赖**：如果需要新增 JS 文件或 CSS 类，先在群里说明用途
3. **数据模型变更**：Storage 的字段结构如有调整，必须同步更新 `docs/开发规范.md` 中的对应数据模型描述
4. **阻塞求助**：遇到技术问题 30 分钟内解决不了的，直接群里说，不硬抗
5. **每日同步**：每天晚上报进度——完成了哪个步骤、明天准备做什么

## 四、关键约束

| 约束 | 说明 |
|------|------|
| 8MB 包体 | 初赛 Demo 总包体不超过 8MB |
| 无网络 | 纯本地运行，不能依赖 API 调用 |
| 纯 JS | 不能使用框架（React/Vue 等），不能使用构建工具 |
| 图片限制 | 用户照片用 JPEG quality=0.6，最长边 800px；纹理优先用 CSS 绘制 |

## 五、模块接口一览（重要）

两个模块共享这些接口，任何人**不要修改已有方法的签名**：

```
Storage:
  getLetters() → Letter[]
  saveLetter(letter) → Letter
  addEcho(letterId, type) → echoesObject
  getEchoesForLetter(letterId) → { stone:n, leaf:n, ... }
  getTimeLetters() → TimeLetter[]
  saveTimeLetter(tl) → TimeLetter
  unlockTimeLetter(id, photo) → TimeLetter
  getDrafts() → Draft[]
  saveDraft(draft) → Draft
  deleteDraft(id) → Draft[]
  getUserStats() → { lettersSent, echoesGiven, timeLettersCreated }
  incrementStat(key) → UserStats

WallEngine:
  getCurrentTimeState() → { name, hue, sat, light, desc }
  renderPublicWall(container, letters, onClick)
  renderTimeBoard(container, timeLetters, onClick)
  renderDrawer(container, myLetters, drafts, onLetterClick, onDraftClick)

LetterComponent:
  SLOT_CONFIG / PAPER_CSS / ECHO_EMOJI
  renderEnvelope(letter, onClick) → HTMLElement
  renderContent(letter, echoes) → HTML字符串
  renderEchoPicker(onPick) → HTML字符串
  escapeHtml(text) → string
  formatDate(ts) → string

Camera:
  takePhoto() → Promise<string> dataURL
  extractFeatures(dataURL) → Promise<object>
  compareFeatures(fA, fB) → number (0~1)
  makeFadedPhoto(dataURL) → Promise<string>
```
