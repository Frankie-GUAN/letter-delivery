# 「未曾寄出的信」— 开发规范

> 抖音AI创变者大赛 · 赛道一 · 互动空间（轻量小游戏）
> 初赛 Demo 约束：8MB 包体 / 无网络通信 / 纯本地运行

---

## 一、项目概述

### 1.1 产品定义

一个以"墙"为载体的情感互动作品。用户将没说出的话写成信，藏在墙缝、信箱、窗台或门缝下，等待陌生人的偶然发现。同时，用户可以将特定信件绑定到真实地点和时间，只有指定好友在正确的时间、站在正确的位置拍照，才能解锁。

### 1.2 核心气质

**安静、有细节、具体**。不是鸡汤平台，不是一个"写信工具"。是"你路过的一面墙，墙上有人在说话"。AI 的角色不是评判者——是保管者、观察者、无声的见证人。

### 1.3 三个模块

| 模块 | 一句话 | 玩家数 |
|------|--------|--------|
| **公共墙** | 匿名发现陌生人的信，留下无声回响 | 单人 |
| **私人抽屉** | 管理自己寄出的信和徘徊的草稿 | 单人 |
| **时光信** | 在地点埋信、设时间锁、好友打卡解锁 | 单人+社交 |

## 二、技术架构

### 2.1 技术选型

- **纯 HTML/CSS/JS** — 无框架，无构建工具（Vite/Webpack等），适配抖音互动空间
- **单文件或多文件结构** — 入口 `index.html` 通过相对路径引用本地 JS/CSS 资源
- **Canvas API** — 照片特征提取与褪色处理
- **localStorage** — 数据持久化（Demo 阶段）
- **Camera API** — 拍照（`<input type="file" accept="image/*" capture="environment">`）
- **纯离线运行** — 所有资源包含在项目目录内，零外部依赖

### 2.2 硬约束（必须遵守）

| 约束 | 说明 | 违反后果 |
|------|------|----------|
| **无网络请求** | 禁止使用 `fetch`、`XMLHttpRequest`、`axios`、`WebSocket` 等任何形式的网络 API | 初赛淘汰 |
| **无外部资源** | 禁止引用任何 CDN 或外部服务器上的资源（CSS/JS/字体/图片等） | 初赛淘汰 |
| **无外部跳转** | 禁止 `<a>` 标签外链、`window.location`、`window.open` 等跳转行为 | 初赛淘汰 |
| **8MB 包体** | 最终 zip 包体积小于 8MB | 初赛淘汰 |
| **相对路径** | 所有资源引用使用 `./` 相对路径，如 `src="./js/app.js"` | 加载失败 |
| **竖屏适配** | 内容界面动态适配各种尺寸移动端屏幕，无横向滚动条 | 体验扣分 |
| **错误兜底** | 全局 try-catch + 错误页面，文案："哎呀，出错了，请重启试试吧~" | 体验扣分 |
| **合规内容** | 不含违法违规、IP 侵权、不利于未成年人的内容 | 取消资格 |

### 2.3 数据流

```
用户交互 → 模块控制器 → Storage 层读写 localStorage
                            ↑
                      Letter / Wall Engine 渲染 UI
```

- **单向数据流**：模块读 Storage → 渲染 UI → 用户操作 → 写回 Storage → 重新渲染
- **无事件总线**：模块间不直接通信。唯一的共享数据经由 Storage 层
- **共享层（shared/）** 提供渲染工具，模块负责组装业务逻辑

### 2.4 模块接口约定

每个模块必须暴露以下接口供 `app.js` 调用：

```js
ModuleName.init(...elements)   // 接收 DOM 元素引用，完成初始化
ModuleName.show()              // 切换到该模块时调用
ModuleName.hide()              // 离开该模块时调用
```

公共墙额外暴露：

```js
PublicWall.startWriting()      // 外部触发写信流程
```

时光信额外暴露：

```js
TimeLetter.startCreating()     // 外部触发创建时光信
```

## 三、数据模型

### 3.1 信件 (Letter)

```json
{
    "id": "L_1717000000_a1b2c3",
    "type": "public",
    "content": "信的正文，最长 800 字",
    "paperType": "notebook",
    "slot": "mailbox",
    "signature": "一个也在等雨停的人",
    "sentiment": "warm",
    "createdAt": 1717000000000
}
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | 格式 `L_时间戳_随机6位` |
| type | string | ✅ | `"public"` 公共信 / `"private"` 仅自己可见 |
| content | string | ✅ | 最长 800 字符 |
| paperType | string | ✅ | `"notebook"` / `"receipt"` / `"napkin"` / `"calendar"` / `"burned"` |
| slot | string | ✅ | `"mailbox"` / `"door-gap"` / `"window"` / `"pipe"` / `"wall-crack"` |
| signature | string | ✅ | 落款，不能是真名 |
| sentiment | string | ❌ | 预设信件专用：`"grief"` / `"warm"` / `"secret"` / `"soliloquy"` |
| createdAt | number | ✅ | 时间戳 |

### 3.2 回响 (Echo)

存储结构：`{ [letterId]: { stone: n, leaf: n, lamp: n, dandelion: n, drop: n } }`

| 类型 | 符号 | 含义 |
|------|------|------|
| stone | 🪨 | "我承得住" |
| leaf | 🍃 | "会过去的" |
| lamp | 💡 | "我能看见" |
| dandelion | 🪽 | "去吧" |
| drop | 💧 | "我也是" |

### 3.3 时光信 (TimeLetter)

```json
{
    "id": "TL_1717000000_a1b2c3",
    "content": "信的内容",
    "signature": "署名",
    "photo": "原始照片 dataURL",
    "fadedPhoto": "褪色后照片 dataURL",
    "features": { "tl": 180, "tr": 120, "bl": 90, "br": 150, "center": 130, "histogram": [0.1, 0.3, 0.3, 0.2, 0.1] },
    "openTime": 1735689600000,
    "createdAt": 1717000000000,
    "unlocked": false,
    "unlockPhoto": null,
    "unlockedAt": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| features | object | 照片视觉特征（5区亮度 + 5桶亮度直方图） |
| openTime | number | 时间锁（时间戳，必须 > 当前时间） |
| unlocked | boolean | 是否已解锁 |
| unlockPhoto | string\|null | 解锁时拍的照片 dataURL |
| fadedPhoto | string | 分享卡用的褪色预览图 |

### 3.4 草稿 (Draft)

```json
{
    "id": "D_1717000000_a1b2c3",
    "content": "未完成的内容",
    "paperType": "notebook",
    "slot": "mailbox",
    "signature": "",
    "updatedAt": 1717000000000
}
```

### 3.5 用户统计 (UserStats)

```json
{
    "lettersSent": 5,
    "echoesGiven": 12,
    "timeLettersCreated": 2
}
```

## 四、模块开发规范

### 4.0 共享层接口速查

#### Storage

```js
// 信件
Storage.getLetters()                              // → Letter[]
Storage.saveLetter(letter)                        // → Letter
Storage.getLetterById(id)                         // → Letter | null

// 回响
Storage.addEcho(letterId, type)                   // → echoesObject  (type: stone|leaf|lamp|dandelion|drop)
Storage.getEchoesForLetter(letterId)              // → { stone:n, leaf:n, ... }

// 时光信
Storage.getTimeLetters()                          // → TimeLetter[]
Storage.saveTimeLetter(tl)                        // → TimeLetter
Storage.unlockTimeLetter(id, photoDataUrl)        // → TimeLetter

// 草稿
Storage.getDrafts()                               // → Draft[]
Storage.saveDraft(draft)                          // → Draft
Storage.deleteDraft(id)                           // → Draft[]
Storage.getDraftById(id)                          // → Draft | null

// 统计
Storage.getUserStats()                            // → { lettersSent, echoesGiven, timeLettersCreated }
Storage.incrementStat(key)                        // → UserStats
```

#### WallEngine

```js
WallEngine.getCurrentTimeState()                  // → { name, hue, sat, light, desc }
WallEngine.renderPublicWall(container, letters, onLetterClick)
WallEngine.renderTimeBoard(container, timeLetters, onClick)
WallEngine.renderDrawer(container, myLetters, drafts, onLetterClick, onDraftClick)
```

#### LetterComponent

```js
LetterComponent.SLOT_CONFIG                      // 藏身方式配置
LetterComponent.PAPER_CSS                        // 信纸CSS映射
LetterComponent.ECHO_EMOJI                       // 回响符号映射
LetterComponent.renderEnvelope(letter, onClick)  // → HTMLElement
LetterComponent.renderContent(letter, echoes)    // → HTML字符串
LetterComponent.renderEchoPicker(onPick)         // → HTML字符串
LetterComponent.escapeHtml(text)                 // → string
LetterComponent.formatDate(ts)                   // → "2026.5.30 14:23"
```

#### Camera

```js
Camera.takePhoto()                               // → Promise<string> dataURL
Camera.extractFeatures(dataURL)                  // → Promise<object> 视觉特征
Camera.compareFeatures(fA, fB)                   // → number (0~1 相似度)
Camera.makeFadedPhoto(dataURL)                   // → Promise<string> 褪色照片
```

### 4.1 模块一：公共墙（PublicWall）

**负责人**：[待分配]

**开发目标：**

1. **墙的渲染**
   - 加载 Storage 中的 `type === 'public'` 信件
   - 首次使用时注入 `PresetLetters.ALL` 到 Storage
   - 调用 `WallEngine.renderPublicWall()` 渲染

2. **读信体验**
   - 点击信封 → 模态层打开 → 显示信件正文 + 落款 + 回响风铃
   - 信件正文下方出现回响选择面板（5个符号按钮）
   - 用户选择回响 → 写回 Storage → 刷新风铃数量 → 短暂动画反馈（emoji飘起后消失）

3. **写信流程**（核心）
   - 点击 FAB 按钮 → 模态层打开 → 写信面板
   - **步骤顺序不可变：** 选藏身方式 → 选信纸 → 写正文 → 填署名 → 寄出
   - 校验：内容不能为空（最长800字）
   - 寄出：生成 letter 对象 → `Storage.saveLetter()` → `Storage.incrementStat('lettersSent')` → 动画（信被塞入对应位置） → 关闭模态 → 刷新墙
   - 写信过程中，未完成的信自动存为草稿：`Storage.saveDraft()`

4. **时间状态**
   - 墙的背景颜色/氛围随 `WallEngine.getCurrentTimeState()` 变化
   - 不同时间段浮现的信件可以不同（AI排期逻辑，由预置信件的sentiment字段辅助）

5. **模态层行为**
   - 所有弹窗使用全局 `#modal-overlay` + `#modal-content`
   - 关闭方式：✕按钮 / 点击背景 / Esc键

**交付标准：**
- [ ] 首次打开能看到至少 10+ 封预置信件散布在墙上
- [ ] 能点击信封阅读完整内容
- [ ] 能选择回响物件，回响数量正确显示
- [ ] 能完整走完写信→寄出流程
- [ ] 寄出的信立即出现在墙上
- [ ] 墙的颜色随当前时间变化（清晨/正午/傍晚/深夜）
- [ ] 模态层开关流畅，无残留状态

**负责人**：[待分配]

**开发目标：**

   - 展示所有 `type === 'public' || type === 'private'` 的信

3. **统计面板**

4. **视觉风格**

- [ ] 统计数字正确

### 4.3 模块三：时光信（TimeLetter）

**负责人**：[待分配]

**开发目标：**

1. **公告栏渲染**
   - 展示所有 `Storage.getTimeLetters()`
   - 已解锁的：褪色照片 + 信纸预览 + ✅可打开
   - 未解锁+时间未到：褪色照片 + 倒计时 + 🔒
   - 未解锁+时间已到：褪色照片 + 可打开提示
   - 布局：类似软木板，卡片用图钉钉住，有轻微旋转

2. **创建流程**（3步）
   - **步骤1**：写正文 + 署名 + 选择打开时间（datetime-local input）
   - **步骤2**：拍照（调用 `Camera.takePhoto()`）
   - **步骤3**：预览（褪色照片 + 信纸缩略 + 倒计时） → 确认 → `Storage.saveTimeLetter()`

3. **解锁流程**
   - 时间未到 → 显示"尚未解锁" + 倒计时
   - 时间已到 + 未解锁 → 显示褪色参考照片 + "拍照解锁"按钮
   - 拍照 → `Camera.extractFeatures()` → `Camera.compareFeatures()`
   - 相似度 >= 60% → `Storage.unlockTimeLetter()` → 跳转"已解锁"视图
   - 相似度 < 60% → 显示角度提示（"右边太亮了，往左转一点"）

4. **已解锁展示**
   - 双照片并置：埋信时的褪色照 vs 解锁时的新照
   - 中间一封信
   - 底部浮现一行 AI 观察：对比两张照片的差异
     - 例如："2026.10.3 下午2点 vs 2027.1.1 上午10点。同一个窗口，冬天的太阳比秋天低一点。窗台上多了一盆植物。"

5. **分享卡生成**
   - 生成一张图片（Canvas 合成）：褪色照片 + "寻宝卡"字样 + 地点提示 + 打开时间
   - 用户可保存到相册或分享给好友

6. **AI 旁白（解锁时）**
   - 对比两张照片的视觉特征差异（光照方向、亮度分布变化）
   - 生成一行观察性文字——不是"恭喜解锁"，而是注意到两张照片的不同
   - 若无网络，使用本地模板语句拼接（例如根据亮度差异选择模板）

**交付标准：**
- [ ] 能创建时光信（完整3步流程）
- [ ] 公告栏正确展示所有时光信
- [ ] 时间锁逻辑正确（未到/已到/已解锁三种状态）
- [ ] 拍照解锁功能可用，相似度判定合理（60%阈值）
- [ ] 解锁后双照片并置展示
- [ ] 角度提示文案合理
- [ ] 分享卡可生成并保存

## 五、预置信件规范

### 5.1 质量要求

每封预设的信必须满足：
- **有具体细节**：一个物件、一个地方、一个时刻、一句对话。不能是"生活很累但要坚持"这种抽象表达
- **有"缺陷"**：句子不需要完整，逻辑不需要清晰。真实的人写信时有停顿、有矛盾、有"算了不说了"
- **不能AI味**：禁止"在这个快节奏的时代"、"我们要学会"、"或许这就是生活吧"等模板句式
- **200~400字**：太长没人读完，太短没温度

### 5.2 情绪分布

目标 20 封信，情绪配比：

| 情绪 | 数量 | 说明 |
|------|------|------|
| grief | 2~3 | 告别/辜负/来不及说的话 — 必须克制，不能煽情 |
| warm | 5~6 | 日常善意/微小发现/暖暖的瞬间 — 必须具体到人事物 |
| secret | 4~5 | 没对当事人说的话/暗恋/秘密 — 有名字更好 |
| soliloquy | 4~5 | 对自己说的话/审视/转折 — 可以笨拙 |

### 5.3 信纸+藏身方式搭配建议

- 急切的道歉 → `napkin` + `door-gap`（仓促塞进门缝的餐巾纸）
- 暗恋/秘密 → `notebook` + `wall-crack`（墙缝里的小纸条）
- 对自己的话 → `calendar` + `pipe`（烤黄的日历页）
- 温暖记忆 → `receipt` + `mailbox`（信箱里的小票背面）
- 告别/释怀 → `burned` + `window`（烧过后从窗口飘进来）

```
<type>: <简短描述>

  feat:  新功能
  fix:   修复Bug

  refactor: 重构
```

2. 推送到 GitHub，发起 PR 到 main
3. 至少一人 Review 后合并
4. **不要直接 push main**

## 八、编码规范

### 8.1 JavaScript

- 每个模块使用 IIFE 或 Revealing Module 模式（参考已建框架）
- 使用 `const` / `let`，禁止 `var`
- 函数名使用 camelCase，常量使用 UPPER_SNAKE
- 每个文件顶部有简要注释说明职责
- 禁止使用 `eval()`、`innerHTML` 直接插入用户内容（使用 `escapeHtml()` 工具函数）

### 8.2 CSS

- 使用 `var(--xxx)` 访问定义好的 CSS 变量（见 main.css），不要硬编码颜色
- 新增配色需求先在 `:root` 中添加变量
- 动画使用 `@keyframes`，时长不超过 1s
- 字体族：中文使用宋体/楷体系列 `var(--font-body)` / `var(--font-hand)`
- 禁止使用外部字体文件（8MB 包体限制）

### 8.3 静态资源

- 所有图标使用 emoji 或 CSS 绘制，不使用图片文件（节省包体）
- 纹理效果（纸张、墙面）优先使用 CSS `background` 渐变/噪点模拟
- 若必须使用图片：JPG 格式，单张不超过 50KB，总量不超过 500KB

### 8.4 性能

- 单次 localStorage 读取的数据量控制在 5MB 以内
- 照片存储使用 JPEG 格式 `quality=0.6`，最长边不超过 800px
- 从读照片到特征提取完成，主线程阻塞不超过 500ms（使用 Canvas 降采样）

## 九、8MB 包体预算

| 类别 | 预算 | 说明 |
|------|------|------|
| HTML + JS | ~500KB | 所有代码文件 |
| CSS | ~100KB | 全局样式 + 动画 |
| 预置信件文本 | ~50KB | 20封信 × 平均300字 ≈ 12KB 原始文本 |
| 静态资源 | ~500KB | emoji 图标 / CSS纹理 / 少量音效 |
| **照片存储（运行时）** | **剩余空间** | 用户拍照/时光信照片占用，需限制照片质量 |

> 注意：运行时 localStorage 有 5-10MB 限制（不同浏览器），照片存储需控制。

## 十、FAQ

**Q: 共享层代码不够用怎么办？**
A: 先在模块内自行实现，如果发现其他模块也需要，再提 PR 迁移到 shared/ 目录。

**Q: 写信面板公共墙和时光信各有一套，怎么复用？**
A: 公共墙的写信面板作为"模板"，时光信可以在其基础上扩展（增加拍照步骤）。如果共享代码，考虑抽到 shared/ 的 letter-writer.js。

**Q: 拍照功能在桌面端怎么测试？**
A: `<input type="file" accept="image/*">` 既支持调用相机（移动端），也支持选择相册图片（桌面端）。桌面端测试时选用相册即可。

**Q: 预置信件谁写？**
A: 建议团队中文功底最好的成员负责。当前已有14封示例，目标至少20封。AI 可以辅助生成，但每封必须人工润色去"AI味"。
